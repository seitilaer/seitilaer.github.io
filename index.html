Nginx 301

Fri, May 28, 2021 在 http://v.com/#/apps 跑 js window.location.href = '/v' 页面跳转到 http://v.com:8080/v/ 由于 v 的末尾没有 /，nginx 会查找有没有 v 文件，有的话就返回，但 v 是文件夹，所以 nginx 会在末尾添加 /，然后是 301 跳转，Location->http://v.com:8080/v/ 浏览器地址的 port 是 80，8080 是 nginx 的 server listen port，我们希望的 Location 是 http://v.com/v/ 找到的方案 把浏览器地址的 port 和 nginx 的 port 保持一致 absolute_redirect off; off 就是 nginx 给出的 redirects 是非 absolute，是 relative，也就是不带 ip、port，那 Location 是 /v/，就会跳转到 http://v.com/v/ port_in_redirect off; off 就是不带 port，那 Location 是 http://v.
